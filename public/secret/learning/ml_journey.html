What is Machine Learning?
Big picture, 3 types of ML, interactive demo, and when to use it
Machine Learning gives computers the ability to learn from examples — instead of writing rules by hand, you feed data to an algorithm and it finds the patterns automatically.

📝 Traditional Programming

You write the rules → Rules + Data → Answers

Example: "If temperature > 30°C AND humidity > 80%, send heat alert." Every rule written by you.

🤖 Machine Learning

You give Data + Answers → Algorithm learns rules → Model predicts new Answers

Example: Show millions of weather readings + outcomes. Model discovers patterns on its own.

💡 ML shines when rules are too complex to write manually — like recognising faces, translating languages, or detecting fraud across thousands of signals.
🏷

Supervised Learning

Learns from labeled examples (input + correct output pairs).

Regression — predict a number (house price, temperature)
Classification — predict a category (spam/not spam)

🏠 House prices 📧 Spam filter 🩺 Cancer detection
🔍

Unsupervised Learning

Finds hidden patterns with no labels given.

Clustering — group similar data (K-Means)
Dimensionality Reduction — compress data (PCA)

👥 Customer segments 🚨 Fraud detection
🎮

Reinforcement Learning

Agent learns via reward & punishment from environment.

Trial and error — maximize cumulative reward over time.

♟ AlphaGo 🚗 Self-driving 🤖 Robotics
💡 90% of real-world ML is Supervised Learning. Unsupervised is powerful for exploration. RL is cutting-edge but harder to apply.

✅ Use ML when...

→ Rules are too complex to hand-code
→ You have lots of labelled data
→ Patterns change over time
→ Performance matters at scale
→ Problem has clear inputs & outputs

❌ Don't use ML when...

→ A simple if/else rule works fine
→ You have very little data
→ You need 100% explainability
→ The problem is static & well-defined
→ Cost of errors is catastrophic

1️⃣ Handle Missing Values

Fill or remove NaN cells.

• Drop rows — safe only if data is plentiful and missingness is random
• Mean/Median imputation — replace NaN with column average (use median for skewed data)
• Mode imputation — for categorical columns, fill with most common value
df.fillna(df.median())  |  df.dropna()

2️⃣ Encode Categorical Variables

Convert text categories into numbers.

• One-Hot Encoding — creates a binary column per category. No artificial ordering.
NYC → is_NYC=1, is_LA=0, is_Chicago=0
• Label Encoding — maps categories to integers. Can imply false order.
pd.get_dummies(df['City'])

3️⃣ Feature Scaling

Bring all features to the same scale.

• Standardization: z = (x − μ) / σ → mean=0, std=1
• Min-Max Normalization: x′ = (x − min) / (max − min) → range [0,1]
⚠️ Always fit scaler on training data only, then apply to test data.
StandardScaler().fit_transform(X_train)

4️⃣ Handle Outliers

Cap, remove, or transform extreme values.

• IQR Method — values beyond 1.5× IQR are outliers
• Log Transform — for right-skewed data: log(income) compresses range
• Capping — clip at 1st/99th percentile
np.log1p(df['income'])

5️⃣ Feature Selection

Remove irrelevant or redundant columns.

Drop: columns with >50% missing, near-zero variance, correlation >0.95, IDs/names
Techniques: Correlation matrix, mutual information, LASSO regularisation
df.corr()  |  SelectKBest(k=10)

💡 A single neuron = linear regression + activation function. Stack thousands in layers and they can approximate any function — the Universal Approximation Theorem.

Data flows left → right through every layer. Each neuron computes its weighted sum, applies ReLU, passes forward. Hit ▶ to animate.

Inactive Activating Active (firing) Suppressed (ReLU=0)
💡 Depth = power. Layer 1 detects edges, Layer 2 detects shapes, Layer 3 detects faces. This hierarchy of representations is what makes deep learning powerful.

Backpropagation computes how much each weight contributed to the error, then nudges all weights to reduce it. Click each step to walk through it.

1 · Forward Pass → Get Prediction

Run input through network, get ŷ. Example: model predicts 0.3, correct answer is 1.

ŷ = forward(x, W)

2 · Compute Loss

Measure how wrong the prediction is. One number summarising all error.

L = (y − ŷ)² = (1 − 0.3)² = 0.49

3 · Compute Output Gradient

How much does loss change if we nudge the output? Derivative of loss with respect to ŷ.

∂L/∂ŷ = −2(y − ŷ) = −1.4

4 · Chain Rule — Propagate Backward

Error flows backward through layers using the chain rule. Each layer gets its gradient.

∂L/∂w = ∂L/∂ŷ · ∂ŷ/∂z · ∂z/∂w

5 · Update Every Weight

Nudge each weight in the direction that reduces the loss. α = learning rate.

w ← w − α · ∂L/∂w
💡 Backprop was the breakthrough of the 1980s. It's just the chain rule from calculus applied cleverly — but it unlocked everything from image recognition to language models.

Without activation functions, a neural network is just linear regression. Activation functions add non-linearity.

Model Evaluation
Bias-variance tradeoff, cross-validation, choosing metrics, and diagnosing learning curves

Too simple → can't learn (high bias). Too complex → memorises noise (high variance). Pick a complexity level.

📉

Underfitting

High Bias · Low Variance
Fix: more complex model

🎯

Sweet Spot

Balanced · Generalises well
Goal: minimise both

📈

Overfitting

Low Bias · High Variance
Fix: regularisation, more data

💡 Total Error = Bias² + Variance + Irreducible Noise. You can't eliminate all three simultaneously. Find the sweet spot for your dataset.

K-Fold Cross-Validation splits data into K chunks, trains K times, and averages scores. Much more reliable than a single split.

Cross-Validation Results
💡 Always use CV when selecting a model or tuning hyperparameters. Use the test set only once — at the very end. Peeking at test data to guide decisions is leakage.

The gap between training and validation curves tells you exactly what's wrong. Pick a scenario.

💡 High training accuracy means nothing by itself. The gap between train and validation performance tells you everything about overfitting.
